# Final 原理理解问题

以下问题用于最终验收，主要检查学生是否真正理解实现原理。建议要求学生用自己的 CPU 代码或波形截图辅助回答。

1. 单周期 CPU 中，`slt` 和 `sltu` 的区别是什么？为什么同一组操作数在有符号和无符号比较下可能得到不同结果？
2. `slti` 和 `sltiu` 的立即数都需要符号扩展吗？为什么 `sltiu` 的立即数不是零扩展？
3. `slli`、`srli`、`srai` 的立即数中哪些 bit 是移位量？为什么通常只使用低 5 位？
4. `srai` 和 `srli` 在处理负数时结果有什么差别？请举一个 32 位数例子说明。
5. `andi`、`ori`、`xori` 的立即数如何扩展？如果立即数是 `0xfff`，实际参与运算的 32 位值是多少？
6. 分支指令的 B-type 立即数为什么最低位固定为 0？硬件里如何还原真实跳转偏移？
7. `beq`、`bne`、`blt`、`bge`、`bltu`、`bgeu` 能否共用同一个 ALU？如果能，ALU 需要输出什么信息？
8. `jal` 写回寄存器的值应该是当前 PC、目标 PC，还是 `PC + 4`？为什么？
9. `jalr` 的目标地址为什么要把最低 bit 清零？如果不清零可能发生什么？
10. 单周期 CPU 中，`jalr` 需要同时使用 `rs1` 和立即数，NPC 模块应该从哪里拿到 `rs1` 的值？
11. 流水线 CPU 中，如果一条分支在 EX 阶段才知道是否跳转，IF/ID 中已经取到的指令应该如何处理？
12. 流水线 CPU 中，`jal` 或 `jalr` 后面的顺序指令为什么可能需要 flush？
13. 连续执行 `addi x1, x0, 1` 和 `add x2, x1, x1` 时，流水线 CPU 如果没有转发或阻塞会发生什么？
14. `lw x1, 0(x2)` 后立刻使用 `x1` 的指令为什么比普通 ALU-ALU 相关更难处理？
15. 数据内存检查比只看寄存器更能发现哪些类型的错误？
16. 为什么最终测试要拆成多个小 `.dat` 文件，而不是只运行一个大排序程序？
17. 如果一个测试点失败，如何根据“寄存器错误”和“内存错误”区分是译码、ALU、访存还是写回阶段的问题？
18. 学号排序程序中的 `swap` 子过程为什么需要 `jal` 和 `jalr` 配合？`x1` 在其中保存了什么？
